## 1. Project Overview

Origin Property Concierge is a premium property management marketing and lead
capture platform for Melbourne investors. It consists of:

- **Public website** — hero, property management landing, knowledge centre
  (property guides / articles), contact, consultation booking, lead forms.
- **Lead capture system** — structured Lead entity with address, property
  details, urgency, and status workflow.
- **Knowledge Centre** — CMS-driven articles with seeded fallback guides for
  SEO resilience.
- **Origin CMS v2** — a frozen editorial publishing system for articles,
  designed to receive content from a future external AI engine ("Hermes Lab").
- **Admin layer** — admin-only article management, admin route guard.

**Design system:** Parchment background (#F9F7F2), midnight text (#1A1C1E),
navy accent (#2C3E50), golden accent (#C4985A). Fonts: Playfair Display
(headings) + Inter (body).

## 8. User Entity (Built-in)

`User` is managed by the Base44 platform — **never created or deleted via the
SDK**. Users join via invites (`base44.users.inviteUser(email, role)`).

**Built-in fields:** `id`, `created_date`, `full_name`, `email`
**Editable field:** `role` (defaults to `'admin'` / `'user'`)

**Security:** Only admins can list, update, or delete other users. The
`AdminRoute` component checks `user.role === 'admin'` before rendering admin
pages.

### hermesImport — Hermes Lab Integration Endpoint

This is the prepared entry point for the future Hermes Lab AI engine. It
accepts a JSON payload and maps it to the Article entity. **No AI generation
occurs inside the CMS or this function** — it only receives and stores
finished content.

**Accepted payload fields:**
`title`, `slug`, `category`, `summary`, `hero_image` (or `hero_image_url`),
`body`, `faq` (or `faq_items`), `seo_title`, `meta_description`,
`focus_keyword`, `search_intent`, `target_audience`, `reading_time`,
`word_count`, `seo_score`, `internal_links`, `external_references`,
`suggested_internal_links`, `suggested_external_references`,
`hermes_metadata`, `status`, `featured`.

**Behaviour:** Creates a new article if the slug doesn't exist, or updates the
existing article with the matching slug.

## 11. Base44-Specific Dependencies & Infrastructure

These are the parts of the codebase that tie the app to the Base44 platform.
**They are the primary refactoring targets if migrating fully off Base44.**

### SDK Client — `src/api/base44Client.js`

```js
import { createClient } from '@base44/sdk';
import { appParams } from '@/lib/app-params';

export const base44 = createClient({
  appId: appParams.appId,
  token: appParams.token,
  functionsVersion: appParams.functionsVersion,
  serverUrl: '',
  requiresAuth: false,
  appBaseUrl: appParams.appBaseUrl
});
```

This pre-initialized client is imported throughout the app as `base44` and
provides:
- `base44.entities.{EntityName}.{list|filter|get|create|update|delete|...}`
- `base44.auth.{me|isAuthenticated|logout|redirectToLogin|...}`
- `base44.functions.invoke('functionName', payload)`
- `base44.integrations.Core.{InvokeLLM|UploadFile|SendEmail|...}`
- `base44.analytics.track({ eventName, properties })`
- `base44.users.inviteUser(email, role)`
- `base44.agents.*` (agent conversations, if used)

### App Parameters — `src/lib/app-params.js`

Resolves runtime config from URL params, localStorage, or Vite env vars:
- `VITE_BASE44_APP_ID`
- `VITE_BASE44_FUNCTIONS_VERSION`

## 14. Migration Notes — VS Code / GitHub

### What Can Migrate Directly

- **All frontend source** (`src/`) — React components, pages, Tailwind styles,
  shadcn/ui. These are standard and framework-agnostic (minus the Base44 SDK
  imports).
- **Static assets** (`index.html`, fonts, images).

  definitions that can be recreated in any database.
- **Seeded guide content** (`src/data/seededGuides.js`).
- **Tailwind config** (`tailwind.config.js`, `postcss.config.js`).

### What Requires Refactoring (Base44 Coupling)

| Component                          | Current                         | Migration Target                          |
| ---------------------------------- | ------------------------------- | ----------------------------------------- |
| SDK client (`base44Client.js`)     | `@base44/sdk` createClient      | Supabase / Firebase / custom API client   |
| Entity operations                  | `base44.entities.X.*`           | Your new DB client (Prisma, Drizzle, etc.)|
| Auth (`AuthContext.jsx`)           | `base44.auth.*`                 | NextAuth / Clerk / custom JWT             |
| Backend functions                  | Deno Deploy + `createClientFromRequest` | Node/Express or serverless functions |
| Integrations (LLM, Upload, Email)  | `base44.integrations.Core.*`    | Direct OpenAI / Resend / S3 calls         |
| Vite plugin                        | `@base44/vite-plugin`           | Remove; replace config injection          |
| App params (`app-params.js`)       | Base44 URL/localStorage params  | Standard `.env` + auth context            |
| Analytics                          | `base44.analytics.track`        | Google Analytics / PostHog / custom       |

### Recommended Migration Order

1. **Export source** — download from Base44 dashboard or use GitHub sync.
2. **Set up Git** — `git init`, commit the export, push to GitHub.
3. **Open in VS Code** — `npm install` to verify dependencies resolve.
4. **Stub the Base44 SDK** — create a compatibility layer that mimics
   `base44.entities.*`, `base44.auth.*`, `base44.functions.invoke()` so the
   frontend compiles while you swap the backend.
5. **Migrate the database** — use the entity JSON schemas (Sections 5–8) as
   DDL blueprints. Recreate tables/collections in your target DB.
6. **Migrate auth** — replace `AuthContext.jsx` with your chosen auth provider,
   keeping the same context shape (`user`, `isAuthenticated`, `logout`).
7. **Migrate backend functions** — port `entry.ts` files to your serverless
   platform. The `hermesImport` function is the most important (it's the
   Hermes Lab entry point).
8. **Replace integrations** — swap `base44.integrations.Core.UploadFile` for
   S3/Cloudinary, `SendEmail` for Resend/SendGrid, `InvokeLLM` for direct
   OpenAI/Anthropic calls.
9. **Remove Base44 Vite plugin** — replace with standard Vite config + env vars.

### CMS Architecture (Already Migration-Ready)

Origin CMS v2 was deliberately designed for portability:
- The `mapHermesPayload()` function in `cmsUtils.js` defines the exact
  Hermes-to-Article field mapping — reusable in any backend.
- The `hermesImport` backend function contains the canonical mapping logic.
- Article export (Markdown + JSON) is built into the editor via `ExportMenu`.
- All Hermes metadata fields are additive and self-documenting.

## 15. Known Limitations After Export

1. **No ZIP download via chat** — the full source must be exported through the
   Base44 dashboard's download / GitHub sync feature (see top of document).

2. **Base44 SDK coupling** — the frontend depends on `@base44/sdk` for all
   data operations, auth, and integrations. A direct `npm install` + run will
   fail without either (a) a running Base44 backend, or (b) a stubbed
   compatibility layer.

3. **Backend functions are Deno-only** — `entry.ts` files use `Deno.serve()`
   and `Deno.env`. They will not run in a Node.js environment without
   porting.

4. **Auth is platform-managed** — login, register, OTP, password reset, and
   session management are handled by the Base44 auth backend. Migrating auth
   requires rebuilding these flows (the UI templates exist in
   `src/pages/Login.jsx` etc. and can be preserved with a new auth provider).

5. **Realtime subscriptions** — `base44.entities.X.subscribe()` relies on
   Base44's WebSocket infrastructure. Not currently used heavily but would
   need a replacement (e.g. Supabase Realtime) if adopted.

6. **User records cannot be imported** — the `User` entity is built-in and
   users can only join via invites. A migration must recreate users in the
   target auth system.

7. **Google Maps features** depend on `GOOGLE_MAPS_API_KEY` being available
   to the backend functions. The key itself is not in the source.

8. **Seeded guides are hardcoded** — `src/data/seededGuides.js` contains
   fallback content embedded in the source. This is intentional for SEO
   resilience but means content updates require a code deploy if the CMS is
   unavailable.

9. **Hermes Lab is not yet connected** — the `hermesImport` endpoint and all
   Hermes-related fields/buttons are prepared but inert. No AI generation
   runs inside the CMS. This is by design.

10. **Visual editing / analytics** — the `@base44/vite-plugin` enables
    in-builder visual editing and analytics tracking. These features are
    lost after migration unless replaced.

## 16. Architecture Decisions & IP Notes

### Design Philosophy

- **Origin CMS = publishing platform only.** It creates, edits, publishes,
  schedules, manages images, handles SEO fields, and runs the editorial
  workflow. It does **not** generate content.
- **Hermes Lab = AI intelligence engine** (future, separate application).
  Hermes will research, analyse competitors, generate articles, SEO, FAQs,
  metadata, internal links, external references, score content, and push
  finished articles into Origin CMS via the `hermesImport` endpoint.
- **Separation of concerns** — the CMS is intentionally kept free of AI logic
  so that the publishing layer remains stable while the AI layer evolves
  independently.

### IP Protection

- All article content, schemas, and the CMS architecture are self-contained
  in the source tree and exportable as Markdown / JSON.
- The `hermesImport` function and `mapHermesPayload()` utility define a
  clean, documented contract between Hermes Lab and Origin CMS — this
  contract is the IP boundary.
- Entity schemas (Sections 5–8) are portable JSON Schema definitions,
  ensuring the data model survives any platform migration.

### Frozen Components (Origin CMS v2.0)

The following CMS sections are frozen — bug fixes only, no redesign:
Article Manager, New Article, Content, Hero Image, Markdown Editor, FAQ,
SEO & Targeting, Internal Links, External References, Hermes Metadata,
Hermes AI, Lifecycle, Version History, Save Article.
